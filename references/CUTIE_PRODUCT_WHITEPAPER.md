## 产品概述

### 产品定位

Cutie是一款专门为P人（感知型人格）设计的个人时间管理软件，采用"人生即剧场"的核心理念，将日常任务管理比作戏剧演出的编排过程。

### 目标用户

- **主要用户**：具有P型人格特质的个人用户
- **使用场景**：个人时间管理、项目规划、日常任务安排
- **核心需求**：灵活的时间安排、最小化元工作、无压力的任务管理

### 产品愿景

创造一个"无需坚持的软件"，让用户即使长时间不使用，也能随时重新开始，没有任何心理负担。

## 核心理念

### "人生即剧场"工作法

> 灵感来源：芙宁娜的戏剧人生观

#### 核心概念

1. **Stage（舞台）**：已经排期的任务（精确到天）
2. **Staging（后台）**：暂未排期或已过期的任务
3. **导演视角**：用户作为导演，安排演员（任务）的演出时间表
4. **演出日程**：精确到点钟的时间安排

#### 工作流程

```
任务创建 → Staging-floating → 项目分类 → Stage排期 → 时间轴执行 → Presence记录
```

### 设计哲学

#### 1. 最小化非必须元工作

- 减少不必要的分类、标签、状态管理
- 自动化能自动化的一切
- 让用户专注于真正重要的决策

#### 2. 无惩罚机制

- 没有红色的"逾期"标识
- 过期任务自然回到Staging
- 支持长期中断后的无缝重启

#### 3. 灵活性优先

- 列表作为筛选器而非文件夹
- 多对多的时间片-任务关系
- 支持临时调整和变更

## 与竞品差异化分析

### 与Sunsama的核心差异

| 功能领域     | Sunsama       | Cutie        | 差异化价值               |
| ------------ | ------------- | ------------ | ------------------------ |
| **任务收集** | Backlog/Inbox | Staging系统  | 更符合P人的思维模式      |
| **时间管理** | 一对一时间片  | 多对多关系   | 避免时间片重叠问题       |
| **完成记录** | 简单完成      | Presence机制 | 记录投入时间而非完成状态 |
| **集成能力** | API集成       | VLM截图导入  | 开源友好的解决方案       |
| **日终仪式** | 日总结        | 自定义仪式   | 个性化的结束体验         |
| **团队功能** | 支持团队      | 纯个人空间   | 专注个人体验             |

### 独特功能优势

#### 1. Staging系统

- **Staging-floating**：替代传统inbox
- **项目分组**：staging-($project_xxx)自动分类
- **筛选视图**：按项目、领域等多维度检视

#### 2. Presence机制

- 记录时间投入而非完成状态
- 避免未完成任务的时间片消失
- 更真实地反映工作状态

#### 3. VLM集成

- 截图识别任务
- AI辅助任务创建
- 无需API配置的智能导入

## 核心功能设计

### 1. 任务管理系统

#### 任务生命周期

```
创建 → Staging-floating → 项目分配 → 时间安排 → 执行 → Presence/完成
```

#### 任务属性

- **基础信息**：标题、描述、创建时间
- **分类信息**：项目、领域（Area）、优先级
- **时间信息**：预估时长、实际时长、截止日期
- **状态信息**：当前阶段、完成进度、Presence记录

### 2. 时间管理系统

#### 时间片设计

- **多对多关系**：一个时间片可包含多个任务
- **无重叠原则**：时间轴上的时间片不能重叠
- **灵活调整**：支持拖拽调整时间和任务分配

#### 时间视图

- **日视图**：详细的时间轴安排
- **周视图**：一周的整体规划
- **月视图**：长期项目概览

### 3. 项目管理系统（V2.0规划）

#### 设计理念

> "个人项目管理需要的仅仅是一张电子纸"

#### 核心特性

- **双向链接式单层块笔记**
- **页面作为块容器**
- **深度系统集成**
- **直接拖拽创建时间片**

### 4. AI辅助系统

#### AI能力范围

1. **自动Area分配**：任务自动分类染色
2. **Staging整理**：项目创建和任务分配建议
3. **子任务生成**：一键生成启动辅助任务
4. **VLM任务识别**：截图智能解析任务

#### AI交互原则

- **用户确认机制**：所有AI操作需用户同意
- **最小干预**：全自动运行，减少人工干预
- **个性化定制**：支持自定义提示词

### 5. 模板和自动化

#### 模板功能

- **快速创建**：常见任务的快速生成
- **时间不固定事件**：灵活的模板应用
- **个人仪式**：小确幸等个性化活动

#### 检查单功能

- **受阻条目**：需要定期检查的任务
- **督促事项**：定期提醒和跟进
- **状态概览**：一次性查看所有需要关注的事项

## AI系统设计

### 系统架构

```
用户输入 → VLM/NLP处理 → 意图识别 → 操作建议 → 用户确认 → 执行
```

### 核心模块

#### 1. 任务分类AI

- **功能**：自动确定任务的Area分类
- **技术**：基于任务描述的语义分析
- **目标**：实现全自动分类，最小化人工干预

#### 2. Staging整理AI

- **功能**：分析Staging中的任务，建议项目创建和任务分配
- **交互**：可呼出AI助手，提供操作建议
- **原则**：所有建议需用户确认

#### 3. 子任务生成AI

- **功能**：为复杂任务生成启动辅助的子任务
- **风格**：默认"少女的絮语"（每个子任务10字以内）
- **定制**：支持用户修改提示词风格

#### 4. VLM截图识别

- **功能**：通过截图识别和导入任务
- **技术**：视觉语言模型处理
- **优势**：无需API集成的智能导入方案

### 配置要求

- **用户自配置**：需要用户填写AI API端点
- **开源友好**：支持多种AI服务提供商
- **隐私保护**：本地处理优先，云端处理可选

## 用户体验哲学

### 核心原则

#### 1. 无需坚持

- **长期中断友好**：一个月不用也能立即重启
- **无心理负担**：没有惩罚机制和负面反馈
- **自然回归**：过期任务自然回到Staging

#### 2. 最小化元工作

- **自动化优先**：能自动化的绝不手动
- **智能分类**：AI自动处理分类和整理
- **直观操作**：拖拽等自然交互方式

#### 3. 个人空间

- **纯个人软件**：不做团队集成功能
- **私密性**：个人的私人空间
- **个性化**：支持个人习惯和偏好定制

### 交互设计原则

#### 1. 直观性

- **拖拽操作**：任务和时间片的直接操作
- **视觉反馈**：清晰的状态和进度指示
- **一致性**：统一的交互模式

#### 2. 灵活性

- **多种视图**：适应不同使用场景
- **快速调整**：支持临时变更和调整
- **个性化**：可定制的界面和功能

#### 3. 效率性

- **快捷操作**：常用功能的快速访问
- **批量处理**：支持批量操作和处理
- **智能建议**：AI辅助的效率提升

## 设计决策记录

### 重要设计决策

#### 1. 为什么选择"人生即剧场"？

- **符合P人思维**：灵活、创意、适应性强
- **降低心理负担**：将任务视为表演，减少压力
- **提供结构感**：在灵活性中提供必要的组织

#### 2. 为什么不做团队功能？

- **专注个人体验**：避免功能复杂化
- **保护私人空间**：维护个人隐私和专注
- **降低开发复杂度**：专注核心功能

#### 3. 为什么不做数据分析？

- **数据失真问题**：个人数据常态失真
- **元工作负担**：保证数据真实性工作繁琐
- **用户体验优先**：专注使用体验而非数据

#### 4. 为什么选择VLM而非API集成？

- **开源友好**：避免商业API依赖
- **用户控制**：用户可选择AI服务
- **创新体验**：截图识别的新颖交互

## 结语

Cutie不仅仅是一个时间管理软件，更是一种生活哲学的体现。我们相信，最好的工具应该是无形的，它应该自然地融入用户的生活，而不是成为额外的负担。

通过"人生即剧场"的理念，我们希望帮助P人找到属于自己的节奏，在灵活性和结构性之间找到完美的平衡。Cutie致力于成为一个真正理解用户需求的软件，一个可以陪伴用户成长的数字伙伴。
