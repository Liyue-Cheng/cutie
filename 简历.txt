=====================================================
【技术架构】
=====================================================

后端技术栈：Rust + Tauri + Axum + SQLite

Feature-Sliced Architecture（特性切片架构），实现高内聚低耦合
SFC（Single File Component）端点设计，每个API端点独立成文件
CABC文档标准（Cutie API Behavior Contract）- 为AI原生开发定制的8段式API规范
自研TransactionHelper + Write Permit写入串行化机制，确保数据一致性
SSE（Server-Sent Events）实现前后端实时状态同步
SQLite + SQLx，完整迁移系统与类型安全

前端技术栈：Vue 3 + TypeScript + Vite + Pinia

创新的硬件启发式前端架构（专利级设计）
5阶段数据处理管线：IF（指令获取）→ SCH（调度）→ EX（执行）→ RES（响应）→ WB（写回）
乱序执行与资源冲突检测，实现3倍性能提升（顺序3秒 → 并行1秒）
状态管理采用RTL硬件设计模式（寄存器/总线/DMA传输）
原子设计组件系统（Parts → Templates → Functional → Alias）
泛型化拖拽策略系统，支持多场景复杂交互

AI原生开发实践特点

0行连续手写代码：所有代码通过AI协作生成
自文档化架构：CABC标准确保完整的8段式文档（签名、行为、I/O、校验、逻辑、边缘、副作用、契约）
模块化与声明式设计：专为AI上下文理解优化
快速迭代：平均12次提交/天，峰值60次/天
